

## functional requirements


What a system must have :
- features
- behaviour

Example :
- User Authentication / Authorization : User can register, log in, and manage accounts
- Core Functionality : Create post, search items or send message
- Data Processing : Storing, retrieving, and updating data
- Reporting : Dashboard, analytics and Notifications.



## Non functional requirements

How system performs ?
Like :
- **Scalability:** The system must support 1 million concurrent users.
- **Availability:** The system must have 99.99% uptime (high availability).
- **Performance:** API requests must return in under 200 milliseconds.
- **Security:** Data must be encrypted at rest and in transit.
- **Usability/Accessibility:** The UI must be intuitive and comply with accessibility standards.
- **Maintainability/Flexibility:** The system should be easy to update without downtime.
