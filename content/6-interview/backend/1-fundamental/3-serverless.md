

1. What is serverless architecture ?
	1. Cloud computing model where developer runs server without managing infrastructure.
	2. Works on event driven architecture. Serverless functions runs on trigger of events.
	3. Scaling and server management is handled by cloud providers like AWS, GCP.
	4. Independently and specifically functions can be scale.
	5. Pay As You Go pricing : cost only for usage.

2. Problems with serverless arch : 
	1. Stateless.
	2. Cold start.
	3. vendor lock-in
	4. Connection pooling in db.

3. What are serverless functions ?
	1. They are stateless functions which are deployed in serverless architecture.
	2. Runs on trigger of event.
	3. Provides autoscaling, independent scaling, cost efficiency.
