- A **Docker volume** is a mechanism for persisting data generated by and used by Docker containers.
- Ensuring that the data is not lost when the container is stopped or removed.
- data can be shared between containers, preserve data when a container is deleted, and manage data independently of the container lifecycle.

**Commands**

| **Command**                                                                                               | **Description**                                       |
| --------------------------------------------------------------------------------------------------------- | ----------------------------------------------------- |
| `docker volume create <volume_name>`                                                                      | Creates a new volume.                                 |
| `docker volume ls`                                                                                        | Lists all volumes.                                    |
| `docker volume inspect <volume_name>`                                                                     | Displays details about a volume.                      |
| `docker volume rm <volume_name>`                                                                          | Removes a specific volume.                            |
| `docker volume prune`                                                                                     | Removes all unused volumes.                           |
| `docker run -v <volume_name>:<container_path> <image>`                                                    | Mounts a volume to a container.                       |
| `docker run --mount type=volume,source=<volume_name>,target=<container_path> <image>`                     | Another way to mount a volume (more explicit syntax). |
| `docker volume create --driver local --opt type=none --opt device=<host_path> --opt o=bind <volume_name>` | Binds a host directory to a volume.                   |
| `docker inspect <container_id>`                                                                           | Shows volume mounts for a specific container.         |



## Two types of volumes:

1. **Named Volumes:**  (recommanded)
	- managed by docker and stored in host machine. 
	- Has a unique name.
	- mapped to a specific location in container, eg `/data` during container creation time.
	- can be mount to multiple conatiners.
	- can be automatically created with `-v ` tag.
create:
```bash
docker volume create my-database-volume
```
mouting:
```
docker run -v my-database-volume:/data --name my-container my-image
```


2. **Bind Mounts:** (used in dev mode)

	- used to bind a specific dir on the host machine to the dir inside the container.

```bash
docker run -v /path/on/host:/path/in/container --name my-container my-image
```

#### Attaching host volumes

Attaching host volumes in Docker allows you to mount a directory from your host machine into a container.
- Change made from one system reflects in other system also.

```
docker run -it --rm -v <host-path>:<conatiner-machine-path> ubuntu  
```

```
docker run -it --rm --mount type=bind,source=/home/use/data,target=/app/data ubuntu
```

##### Readonly : Prevent the container from modifying.

```
docker run -v /home/user/data:/app/data:ro nginx
```

```
docker run --mount type=bind,source=/home/user/data,target=/app/data,readonly nginx
```



#### Custom Volume : 

```
docker volume create my-volume
```

```
docker run -it --rm -v my-volume:<conatiner-machine-path> ubuntu  
```

```
docker run -it --rm --mount ,source=my-volume,target=/app/data ubuntu
```
